9 Keep in mind, gdb is NOT thread safe. Thus we can only safely run gdb.*
10 functions in one thread/process. Nevertheless, there is quite a bit of
11 processing to do in some cases, and some events run long (e.g.
12 extracting/dispatching large sections of memory). We don't want to "stop the
13 world" every time we find a large object, so we *do* use multiple processes
14 (and even threads, but only in one very specific case).
15 
16 The overall motivation for the message passing architecture is simple:
17 I have experimented with multiple ways of extracting information
18 from the inferior process, and this is the only one which meets my design
19 goals.
20 
21 1. The runner (and all other portions of MO) should be as modular as possible.

27 
28 3. The runner should be as simple as possible. Again, this is fairly obvious.

211  def collect(self):
212  while True:
213  inbound = list(self._publishers.values())
214  for p in multiprocessing.connection.wait(inbound):
215  try:
216  msg = p.recv_bytes()
217  self._holding.put(msg)
218  except EOFError:

221  def broadcast(self):

225  for s in outbound:
